# 詳細仕様ガイド

`.litespec/specs/` には、機能単位の詳細仕様を配置します。
`SPEC.md` をインデックスとして保ち、詳細はここに分割して管理します。

## 命名ルール

- `NNN-機能名.md` の形式を推奨（例: `001-auth.md`）
- 番号は追加順に採番し、既存番号は極力変更しない

## 最低限の運用

1. 新機能を追加する場合は、新しい詳細仕様ファイルを作る
2. `SPEC.md` の「詳細仕様一覧」にリンクを追加する
3. 実装後に仕様と実装の差分を確認する

## テンプレート

- 新規作成時は `0000-template.md` をコピーして使う
